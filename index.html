Welcome to the CS633 Automated Testing and Code Quality Page

Modern software development teams increasingly rely on automated testing and code quality tools as fundamental components of their development practices. Through the strategic implementation of unit, integration, and user interface tests, combined with static analysis platforms like SonarQube and Veracode, development teams can achieve several critical objectives:

  • Develop software with measurably higher quality and reliability
  • Identify and resolve defects closer to their point of origin
  • Build confidence in refactoring code through comprehensive regression testing
  • Maintain high development velocity while ensuring quality standards

Throughout this course, you will engage with practical assignments that demonstrate these concepts in action. As you progress through the lessons and hands-on exercises, you will gain firsthand experience with implementing automated testing strategies and code quality practices, developing a deep understanding of their value in contemporary software development.

In this lesson, you will use the JavaScript testing framework Mocha and the BDD / TDD assertion library Chai to execute in-browser tests for a sample rock-paper-scissors application. To begin, fork the project on GitHub to create a personal copy of the repository. You can then clone it locally or use GitHub Codespaces for a browser-based environment, which requires no local installation. For detailed setup and execution instructions, please consult the README.md file in the repository.
